Overview

Welcome! Here you'll find tutorials to help you become a ChiraKit pro. We recommend starting with tutorials 1 and 2. Tutorials 3 to 10 can be completed in any order.

1) Basic manual processing of protein and buffer spectra
  1. Download this zip file which contains the files R78906.d01, R78906.d02, R78906.d03, R78907.d01, R78907.d02 and R78907.d03 and extract them onto your computer. The files are CD measurements of a protein sample ('R78907') and the corresponding reference buffer ('R78906'), kindly provided by Nyk Jones from Aarhus University, Denmark.
  2. In the "Input" section of the "Import data" page, import the six files simultaneously (drag and drop where it says 'Browse', or click 'Browse' and select them).

  3. If the upload is successful you will see the data plotted, with scans shown in different colours.

  4. The increase in the noise level at the lowest wavelengths is due to the high absorbance of the sample in this region, so this data should be excluded. Navigating to the 'HT signal' tab above the figure, you will find a plot of the HT values associated with the CD measurement, a good indicator of the absorbance of the samples measured.

  5. Adjust the 'HT max value' parameter found in the "Processing" section of the page to 6.5. The wavelength range of the plotted data should automatically update to remove the noisy CD data and only plot to 178 nm. This value for "HT max value" is specific to the spectrometer where this test data was measured. For other data it is necessary to use the value as defined by the manufacturer of the spectrometer used for data collection.


  6. Switch to the 'CD signal / working units and HT' tab above the figure which shows a plot of both the CD and HT for all spectra.

  7. In section '4. Plotting', deselect the files for the reference baseline scan, those named R78906.d0x, leaving only the three protein measurements selected (R78907.d0x). The plot will update and now only show the sample protein spectra.

  8. In the section '3. Processing', set the following options:
    Operation: 'Average'
    Spectrum/a 1: 'Selected in Show Column'
    Operation Units: 'Millidegrees'
    Output name: 'sample average'

  9. Click 'Apply' to execute the processing step and press the submit button in the pop-up window to proceed with the operation.

  10. The averaged data will appear as a new curve on the figure and as an entry in the plotting list.

  11. The same procedure needs to be carried out to produce an average of the reference baseline scans, therefore: Deselect the three sample spectra and the "average sample" from the list in the "Plotting" section and select the three reference baseline spectra (R78906.d0x). The figure will now show the three reference baseline scans.

  12. In the section '3. Processing', set the following options:
    Operation: 'Average'
    Spectrum/a 1: 'Selected in Show Column'
    Operation Units: 'Millidegrees'
    Output name: 'buffer average'
  13. Click 'Apply' to execute the processing step and press the submit button in the pop-up window to proceed with the operation.

  14. Subtract the buffer baseline from the sample spectrum by using the following options in the ' Processing' section:
    Operation: 'Subtract'
    Spectrum/a 1: 'sample average'
    Operation Units: 'Millidegrees'
    Spectrum 2: 'buffer average'
    Output name: 'baseline subtracted'

  15. Press apply and then confirm in the pop-up window to carry out the subtraction.

  16. Switching to the "CD Signal / working units" tab on the figure and selecting only the average and subtracted entries in the plotting list, the plot will show only these processed data.

  17. To now convert these data from the units of mdeg into normalised units of molar extinction (Delta epsilon), do the following. In section 2 on the page "Parameters for molar ellipticity / extinction" find the entry for the "baseline subtracted" dataset (note that the raw data files can now be removed in the "input" area of the page, in order make it easier to find and use the treated data in section 2).
  18. Input the parameters of the 'baseline subtracted' spectrum: Set the concentration to 1.3 mg/ml, path length to 0.1 mm, #Chromophore units (this is the number of peptide bonds, so total number of residues minus 1) to 128, and molecular weight to 14300 Da.

  19. To view the result of this conversion in the plot, it is necessary to change the "Working units" from mdeg to "Mean unit molar extinction in the "Processing" section of the page.

  20. The final baseline subtracted spectrum will now be shown in the "CD signal / working units" plot, while the original unconverted data can still be viewed in the "CD signal / millidegrees" tab.

  21. Go to the 'Export data' panel and export the "baseline subtracted" spectrum in delta epsilon units by selecting it from the pull down list and choosing "Specific curve(s)".

  22. A file dialogue window will open prompting the saving of the output text file containing the spectral data selected.

  23. In order to know how the data was processed, the logbook file should also be downloaded and saved with the processed spectrum.

  24. Congratulations! You have successfully processed CD measurements, obtaining a baseline-subtracted spectrum in delta epsilon units. Note that the exported file contains also temperature data which was read from the metadata of the original input files.
2) Basic automatic processing of protein and buffer spectra
  1. Download this zip file which contains the files R78906.d01, R78906.d02, R78906.d03, R78907.d01, R78907.d02 and R78907.d03 and extract them onto your computer. The files are CD measurements of a protein sample ('R78907') and the corresponding reference buffer ('R78906'), kindly provided by Nyk Jones from Aarhus University, Denmark.
  2. Set the 'Input mode' to 'Automatic baseline subtraction'. Upload the 'R78907' files as 'Sample files' and the 'R78906' files as 'Baseline files'.

  3. Click on the 'Process data' button. In the pop-up window that appears, set 'millidegrees' as input units and 'mean unit molar extinction' as desired units of the final spectrum.

  4. If you want to zero the data at long wavelengths, check the box next to "Subtract the average signal of the highest 10 nm wavelength interval from the final CD spectrum?"
  5. In the next pop-up window, enter the required experimental details.
    Path length: 0.1 mm
    Molecular weight: 14300
    Concentration: 1.3 mg/ml
    Number of chromophore units (peptide bonds): 128

  6. Press 'Submit' and wait a few seconds. The finalised spectrum should appear.


  7. Adjust the 'HT max value' parameter to 6.5. The wavelength range should automatically update (to 178 - 330 nm).

  8. Go to the 'Export data' panel and export the "baseline subtracted" spectrum in delta epsilon units by selecting it from the pull down list and choosing "Specific curve(s)".

  9. A file dialogue window will open prompting the saving of the output text file containing the spectral data selected.

  10. In order to know how the data was processed, the logbook file should also be downloaded and saved with the processed spectrum.

  11. Congratulations! You have successfully processed CD measurements in an automated fashion, obtaining a baseline-subtracted spectrum in delta epsilon units.
3) Analysis of protein melting curves using a single-wavelength approach
  1. Download the file thermalRampChirascan.csv. This file contains the thermal denaturation experiment data of a protein sample measured using a benchtop CD instrument.
  2. Import the file into ChiraKit, in "Custom" input mode. Set the wavelength range to 210 - 280 nm to remove noisy data.

  3. Open the ‘Thermal unfolding’ panel (Section '2. Analysis').

  4. Once inside the ‘Thermal unfolding’ panel, on the ‘2. Fitting’ section press on '2a. Create dataset'. Then, press on '2c. Find wavelength'.

  5. Explore how the CD signal changes according to the different proposed wavelengths. Then set the 'Selected wavelength(s)' to '220-226'.

  6. Select the 'N ⇌ U' (Equilibrium Two-State) unfolding model, activate the 'Fit unfolded slope' option and fit the thermal unfolding curves by clicking on '2d Fit data'.


  7. Check the "Fitted Parameters" tab to analyze the numerical results of the fit.

  8. Congratulations! You have successfully fitted a reversible two-state unfolding model to obtain the melting temperature of a protein sample.
4) Singular value decomposition (SVD)-based analysis of protein melting curves

In this section, we will analyze the same dataset as in Tutorial #3 using Singular Value Decomposition (SVD). SVD helps reduce noise and utilize the entire spectral range by decomposing the spectra into a set of basis spectra and their corresponding coefficients.


  1. Download the file thermalRampChirascan.csv. This file contains the thermal denaturation experiment data of a protein sample measured using a benchtop CD instrument.
  2. Import the file into ChiraKit in "Custom" input mode. Set the wavelength range to 210 - 280 nm to remove noisy data.


  3. Open the ‘Thermal unfolding’ panel (Section '2. Analysis').

  4. Press on '2a. Create dataset'. Then change the analysis type (2b) to 'Spectra decomposition (SVD)'.


  5. Click on '2c. Decompose spectra'. Explore the tabs 'Basis spectra', 'Reconstructed spectra', 'Explained variance' and 'Coefficients' tabs.

  6. The first acquired spectrum (20°C) appears to be an outlier. To remove it, right-click on the dataset labeled "thermalRampChirascan 20.0" in the "1. Temperature Data" table and select "Remove row." To proceed with the analysis, create a new dataset by clicking the "Create dataset" button (2a), choosing "Spectra decomposition (SVD)" as the Analysis type (2b), and decomposing the spectra again by clicking the "2c. Decompose spectra" button.


  7. Select the 'N ⇌ U' ('Equilibrium Two-State') unfolding model, activate the 'Fit unfolded slope' option and fit the thermal unfolding curves by clicking on '2f Fit data'.



  8. Congratulations! You have successfully 1) applied the SVD algorithm to find a basis spectrum, and 2) fitted a reversible two-state unfolding model (N ⇌ U) to obtain the melting temperature of a protein sample.
  9. Extra step: compare the obtained parameters with the ones from the Tutorial #3. Consider the robustness of each approach. Tip: when comparing samples, always use the same analysis method.
5) Calculation of secondary structure with Selcon3

In this section, we will estimate the secondary structure content of two well-studied proteins using the Selcon3 method. The Selcon3 method is a self-consistent procedure for estimating the secondary structure content of proteins. It compares the experimental CD spectrum with a set of reference spectra to estimate the percentage of secondary structure elements.


  1. Download the file Myoglobin.csv and OpcA.csv. These files contain the spectra in delta epsilon units (Mean unit molar extinction) of Myoglobin and OpcA. The data was downloaded from the Protein Circular Dichroism Data Bank (Ramalli, Sergio Gomes et al., 2022). The CD spectrum of Myoglobin was published in Lees, Jonathan G. et al., 2006. The CD spectra of OpcA and Wheat germ agglutinin were published in Abdul-Gader, Ali et al., 2011.
  2. Import the two files simultaneously. Set the 'Input units' and the 'Working units' to mean unit molar extinction'.


  3. Navigate to the 'Protein secondary structure' panel (Section '2. Analysis').

  4. Set the 'Analysis mode' to 'SELCON'. Adjust the 'Lower WL' parameter to 180. As a result, the AU-SMP18 reference dataset, comprising soluble and membrane proteins, will be used. Press on 'Run estimation!'.

  5. You should be able to see the percentage of each secondary structure element in the ‘Secondary Structure estimation’ tab and the fitted spectra in the respective tab.


  6. Congratulations! You have successfully estimated the percentage of secondary structure elements using the Selcon3 algorithm and the AU-SMP180 reference dataset.
  7. Extra step: Upload the PDB files 1a6m.cif (Vojtěchovský, Jaroslav et al., 1999) and 2vdf.cif (Cherezov, Vadim et al., 2008). Compare the estimation to the actual secondary structure elements.

7) Calculation of secondary structure with SESCA
  1. Download the file Myoglobin.csv. This file contains the spectra in delta epsilon units of Myoglobin. The data was downloaded from the Protein Circular Dichroism Data Bank (Ramalli, Sergio Gomes et al., 2022). The CD spectrum of Myoglobin was published in Lees, Jonathan G. et al., 2006.
  2. Import the file. Set the 'Input units' and the 'Working units' to mean unit molar extinction'.
  3. Navigate to the 'Secondary structure estimation' panel (Section '2. Analysis'). Set the 'Analysis mode' to 'SESCA (estimation)'. The default basis set is 'DS-dT', which contains three different structural elements and does not consider side-chain contributions. Set 'Myoglobin' as the reference spectrum and 250 as the number of iterations. Press on 'Run estimation!'.
  4. After some minutes, you should be able to see the percentage of each secondary structure elements and posterior probabilities of the different structural elements. The percentage of alpha-helix should be around 70-80%.

  5. We can compare the estimated secondary structure components with the actual secondary structure composition present in the protein structure, and with the predicted spectrum. Set the 'Analysis mode' to 'SESCA (prediction)'. Download the PDB file 1a6m (https://www.rcsb.org/structure/1a6m) and import it. Set Myoglobin as the reference spectrum. Press on 'Run prediction!'. In this case, the almost perfect match is probably explained by the facts that 1) the reference spectrum was part of the SESCA training set, and 2) the secondary structure content is mainly alpha.

  6. Congratulations: you have successfully estimated the percentage of secondary structure elements using the SESCA algorithm. Some extra tips: For beta-rich proteins, we recommend the HBSS (or HBSS-3SC1) basis set. For proteins with a high percentage of turns, we recommend the DS5-4 (or DS5-4SC1) basis set.
8) Analysis of a protein dimer chemical denaturation curve
  1. Download the file trp_repressor_urea_unfolding.csv. The data corresponds to the chemical denaturation of the Trp repressor by urea, monitored by circular dichroism, as published by Andrew N. Lane and Oleg Jardetzky (1986). The data was digitised from Figure 3 of the paper.
  2. Inside the '1. Input' section, choose 'Processed data (chemical)' as the input mode and import the file.

  3. Navigate to the 'Chemical unfolding' panel (Section '2. Analysis').

  4. You should observe a chemical denaturation curve with two transitions. Therefore, the dimer unfolds via an intermediate state.

  5. In the original paper, two models were tested: one with a dimeric intermediate ('N2 ⇌ I2 ⇌ 2U'), and one with a monomeric intermediate ('N2 ⇌ 2I ⇌ 2U'). These models were referred to as Model II and Model III, respectively. Let's start with Model III by selecting the corresponding model and pressing on the button '2d. Fit data'.

  6. Select the 'Fitted curves' tab to view the fitted curves. They match the experimental data, right?

  7. Select the 'Relative errors' tab to view the relative errors of the fitted parameters. You should notice that the errors are extremely high for the D50step1 and M1 parameters. This indicates that Model III may not be adequate.

  8. Now select the Model II and press on the button '2d. Fit data'.

  9. You probably got an error because the initial estimates are not good enough. To help the fitting algorithm, set the initial guess of D50 to 4. Then, press on '2d. Fit data'.


  10. Check the 'Fitted parameters' tab and the 'Relative errors' tab to analyze the numerical results of the fit. The fitted parameters should have lower relative errors than in Model III, indicating that Model II is more appropriate. Indeed, the authors of the original paper concluded that the Trp repressor unfolds via a dimeric intermediate after performing more experiments.
  11. Congratulations! You have successfully fitted reversible three-state chemical unfolding models to obtain the unfolding parameters of a protein dimer.
9) Analysis of peptide helicity

This tutorial contains two sections. In the first section, we estimate the helicity of a peptide using the CD signal at 222 nm. In the second section, we analyse thermal denaturation CD curves of intrinsically disordered peptides with transient helicity. Both sections are based on the work of Zavrtanik, Uroš et al., 2024.


  1. Inside the '1. Input' section, using 'Custom' as input mode, load the file PaaA2_spectrum_25C.csv. Choose 'Mean unit molar ellipticity' both as 'Input units' and as 'Working units'. Ensure that the number of peptide units (#Chromophore units) is specified under the Parameters for molar ellipticity / extinction section. The peptide PaaA2 has 29 peptide units.


  2. Navigate to the Analysis tab and select the 2f. Peptide helicity module. Click on 'Initialize the peptide dataset' to prepare the data for analysis. In the dataset table, input the temperature(s) of the measurement(s). In this case, 25°C. Click on 'Fit peptide helicity' to fit the data using the ensemble model.

  3. The output is estimated helicity reported as a value in the range [0.0–1.0] and Lifson-Roig propagation (w) parameter that indicates overall helical propensity of the peptide.

  4. Now, we'll analyse the thermal denaturation curve of a peptide with transient helicity.
  5. Ensure that 'Processed data (thermal)' mode is selected under the '1. Input' mode section and load the file PaaA2_thermal_data.csv. The first column should contain the temperature array. The second column should contain the processed thermal ellipticity data at 222 nm, represented as the mean unit molar ellipticity. Ensure the unit of ellipticity is deg·cm2·dmol-1·chromophore-1.

  6. Navigate to Thermal unfolding panel (2a) under Analysis tab. Under Unfolding model (Fitting) choose the 'Helix ⇋ Unfolded (Peptide)' model.
  7. Specify the number of peptide bonds (chromophores). For N-acetylated, C-amidated peptides, set the value to Nresidues + 1. For uncapped peptides, set the value to Nresidues - 1. In this case, set the value to 29.

  8. Fit the data by pressing the 'Fit data' button.
  9. After the fitting procedure is successfully completed, the results are displayed in multiple tabs. The data points and fitted curve (solid line) are shown under the 'Fitted melting curves' tab. The fitted parameters are available under the 'Fitted parameters' tab, while the estimated helicity as a function of temperature is displayed under the 'Fractions' tab. In this case, we observe that the disordered fraction does not start at 0. This is in contrast to proteins, where the unfolded fraction typically starts at 0.

  10. Note: Apply this thermal denaturation model only for peptides that adopt helix or coil conformations.
10) Comparison of spectra

  1. Download the file spectra_comparison_example.csv. This file contains artificially generated CD spectra: five measurements of sample 'A' and five measurements of sample 'A' at a different concentration (called 'A2').
  2. Import the file into ChiraKit in "Custom" input mode. Replicated column names will be automatically modified in the Legends Table, under the 'Plotting' section.


  3. You change the plot labels inside the 'Plotting' section. For example, you can set them to 'A1' and 'A2'.

  4. Navigate to the 'Spectra comparison' panel (Section '2. Analysis').

  5. Set the labels to 'A1' and 'A2' and press on '2a. Create dataset'. The average spectra with the associated standard deviations will be shown under the 'Cd avg ± sd' tab.


  6. The difference between the average of 'A1' and the average of 'A2' is shown under the 'Difference spectra' tab. Before 290 nm, there is a clear difference in signal. This is due to the concentration effect.

  7. Click on the '2b. Normalisation' and choose 'L2_norm'. This way, the spectra will be normalised by the L2 norm. You should notice that the difference between the two spectra has disappeared. In other words, the difference previously observed can be explained by a difference in scale (concentration effect). Indeed, now the distances distribution within categories ('Norm A1 versus Norm A1' and 'Norm A2 versus 'Norm A2') overlaps with the distances distribution between categories ('Norm A1' versus 'Norm A2').


  8. Congratulations! You have successfully compared two groups of CD spectra. You can also use this module to compare different conditions, mutants, or replicates.
